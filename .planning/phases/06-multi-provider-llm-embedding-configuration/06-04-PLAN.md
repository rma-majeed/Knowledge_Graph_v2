```python
# Two possible return types from get_llm_client() / get_embed_client():

# 1. LM Studio (default): raw OpenAI client
#    Supports: client.chat.completions.create(model=..., messages=[...])
#    Supports: client.embeddings.create(model=..., input=[...])

# 2. Cloud providers: _LiteLLMConfig with .provider, .model, .api_key, .api_base
#    Callers detect with: hasattr(client, 'provider')
#    Usage: import litellm; litellm.completion(model=client.model, api_key=client.api_key, api_base=client.api_base, messages=[...])

# SIMPLER APPROACH for call sites: use a helper that dispatches based on client type.
# Define _llm_call() and _embed_call() helpers inside each pipeline module:

def _llm_call(client, model: str, messages: list, **kwargs):
    """Dispatch LLM completion call to OpenAI client or LiteLLM based on client type."""
    if hasattr(client, 'provider'):  # _LiteLLMConfig
        import litellm
        return litellm.completion(
            model=client.model,
            api_key=client.api_key,
            api_base=client.api_base,
            messages=messages,
            **kwargs,
        )
    else:  # raw OpenAI client
        return client.chat.completions.create(model=model, messages=messages, **kwargs)

def _embed_call(client, model: str, input_texts: list):
    """Dispatch embedding call to OpenAI client or LiteLLM based on client type."""
    if hasattr(client, 'provider'):  # _LiteLLMConfig
        import litellm
        response = litellm.embedding(
            model=client.model,
            api_key=client.api_key,
            api_base=client.api_base,
            input=input_texts,
        )
        # LiteLLM returns same shape: response.data[i].embedding
        return response
    else:  # raw OpenAI client
        return client.embeddings.create(model=model, input=input_texts)
```

BUT NOTE: src/embed/embedder.py already has `response = client.embeddings.create(model=model, input=texts)` and src/graph/extractor.py has `client.chat.completions.create(...)`. These files take `client` as a parameter from the caller — they do NOT need to be changed if we pass the right client type.

For LM Studio (default), get_llm_client() returns a raw OpenAI client — so existing code that calls `client.chat.completions.create(...)` continues to work unchanged.

For cloud providers, the _LiteLLMConfig object does NOT have .chat.completions.create() — so the dispatch helpers above are needed only IF non-LM Studio providers must work. For Phase 6, the tests only test LM Studio (default) in the pipeline tests — the cloud provider tests are in test_config_providers.py (factory-level tests).

DECISION: For call sites in graph/pipeline.py, query/pipeline.py, and app.py, replace hardcoded OpenAI(base_url=...) with get_llm_client() / get_embed_client(). The returned client (OpenAI or _LiteLLMConfig) is passed through to embedder.py and extractor.py unchanged. For now, add the dispatch helper only in query/pipeline.py where the actual LLM completion call is inline (not delegated). For extractor.py, the caller passes client and extractor calls client.chat.completions.create() — this works for OpenAI client (LM Studio) and will require the dispatch helper for cloud providers. Add the helper to extractor.py call site only if needed.

SIMPLEST correct approach (backward compatible + Phase 6 scope):
1. In each file, replace `OpenAI(base_url="http://localhost:1234/v1", api_key="lm-studio")` with `get_llm_client()` (or `get_embed_client()` for embed calls).
2. In query/pipeline.py, the inline `openai_client.chat.completions.create(...)` and `openai_client.chat.completions.create(..., stream=True)` calls need to use the dispatch helper because they call .chat directly.
3. The `embed_query()` in retriever.py takes `openai_client` — it calls `embed_chunks()` which calls `client.embeddings.create()`. For LM Studio this works. For cloud embed providers, embedder.py needs to handle _LiteLLMConfig too — but that is tested only via test_config_providers.py (mocked), so it will work as-is for Phase 6.
